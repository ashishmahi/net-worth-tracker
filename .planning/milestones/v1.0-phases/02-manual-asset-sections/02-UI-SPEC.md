## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | List row text, form helper text, inline error messages, muted sub-text, labels, block headings |
| Heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 | Section page title (e.g. "Gold", "Mutual Funds"); projected corpus total |
| Display | 24px (`text-2xl`) | 600 (`font-semibold`) | 1.2 | Section total amount (e.g. "₹12,34,567") |

Notes:
- Only 3 sizes are in use: 14px, 20px, 24px. No other sizes are permitted in Phase 2 implementation.
- Only 2 weights are in use: 400 (`font-normal`) and 600 (`font-semibold`). `font-medium` must not appear.
- Section page heading: `text-xl font-semibold` — pattern already present in stub pages.
- Section total: `text-2xl font-semibold` — renders below the page heading.
- Block headings inside Settings and Retirement (e.g. "Gold Prices", "Retirement Assumptions"): `text-sm font-semibold` — bold weight signals heading without adding a size level.
- Muted sub-text (empty state body, SIP sub-rows, footnotes, placeholder descriptions): `text-sm text-muted-foreground font-normal`. Muted color creates de-emphasis without needing a smaller size.
- All monetary amounts use the INR locale format: `₹` prefix, Indian number grouping (1,00,000). Use `toLocaleString('en-IN')` at render.

## Color

All values are CSS custom properties from the Zinc shadcn theme (from `src/index.css`). Do not hardcode hex values.

| Role | CSS Variable | Computed Light Mode | Usage |
|------|-------------|---------------------|-------|
| Dominant (60%) | `--background` | hsl(0 0% 100%) — white | Page background, Sheet background |
| Secondary (30%) | `--card` | hsl(0 0% 100%) with border | Card component surfaces, sidebar |
| Muted surface | `--muted` | hsl(240 4.8% 95.9%) — light grey | Empty state container, read-only value display areas |
| Accent (10%) | `--primary` | hsl(240 5.9% 10%) — near-black | Reserved for: primary Save button bg, Add button bg |
| Destructive | `--destructive` | hsl(0 84.2% 60.2%) — red | Reserved for: Delete button (variant="destructive"), inline save error text |
| Muted foreground | `--muted-foreground` | hsl(240 3.8% 46.1%) — grey | Empty state body copy, placeholder values, section subtitles |

Accent reserved for: Save button (Sheet forms), Add [Item] button (list section header). These are the only two elements that use the primary/accent fill. All other buttons use `variant="outline"` or `variant="ghost"`.

Destructive reserved for: Delete button inside Sheet (single occurrence per open Sheet), inline error message text below Save button.

## Layout Patterns

### List Section Layout (Gold, MF, Stocks, Bank Savings)

```
[Page heading — text-xl font-semibold]
[Section total — text-2xl font-semibold, text-foreground]
                                          [Add {Item} — Button variant="default"]

[Card]
  [List row] [list row] [list row]  ← clickable rows, open Sheet on click
  — or —
  [Empty state]
[/Card]

[Sheet — slides from right, width 400px on desktop, full-width on mobile]
  [SheetHeader]
    [SheetTitle: "Add Gold Item" / "Edit Gold Item"]
  [SheetContent]
    [form fields with Label + Input]
    [inline error — text-sm text-destructive, mt-1]
  [SheetFooter]
    [Delete button — variant="destructive", only shown in edit mode]
    [Save button — variant="default"]
[/Sheet]
```

Section total position: Directly below the page heading (top of content area), before the list card. Rationale: user checks total first, then drills into items.

List row structure:
- Primary text (name/label): `text-sm font-semibold text-foreground`
- Secondary text (value/amount): `text-sm text-muted-foreground font-normal`, right-aligned
- Row: `flex items-center justify-between px-4 py-3 cursor-pointer hover:bg-muted/50 transition-colors`
- Rows separated by `<Separator />` (no outer border between rows, border only on Card)

### Inline Form Layout (Retirement, Settings)

```
[Page heading — text-xl font-semibold]

[Block heading — text-sm font-semibold, mb-4]  ← e.g. "Gold Prices", "Retirement Assumptions"
[grid of Label + Input pairs]
[Separator]

[Next block...]

[Save button — variant="default", self-contained per block]
[inline error — text-sm text-destructive, mt-2]
```

Settings: One Save button per logical block (Gold Prices block has its own Save; Retirement block has its own Save). Not a single global Save. This prevents partial save confusion.

Retirement: NPS and EPF balance fields + Save button + projected corpus display card below.

## Section-Specific Contracts

### Gold Section

Sheet fields (in order):
1. Label: "Karat" — select between 24K, 22K, 18K. Use native `<select>` wrapped in shadcn styling or a simple Button group. Do not use a full Combobox.
2. Label: "Weight (grams)" — Input type="text" inputmode="decimal", placeholder="e.g. 10.5"

List row display:
- Primary: karat label (e.g. "22K Gold") — `text-sm font-semibold`
- Secondary: weight formatted as "N.NN g" — `text-sm text-muted-foreground font-normal`
- Section total: computed as sum of (grams × price_per_gram_for_karat from Settings). Show "₹0" if Settings gold prices are not yet set, with muted footnote "Set gold prices in Settings".

### Mutual Funds Section

Sheet fields (in order):
1. Label: "Platform" — Input type="text", placeholder="e.g. PaytmMoney"
2. Label: "Current Value (₹)" — Input type="text" inputmode="decimal", placeholder="e.g. 1,50,000"
3. Label: "SIP / month (₹)" — Input type="text" inputmode="decimal", placeholder="0 if none"

List row display:
- Primary: platform name — `text-sm font-semibold`
- Secondary: current value in INR format — `text-sm text-muted-foreground font-normal`, right-aligned
- SIP amount shown as sub-row: "SIP ₹X/mo" in `text-sm text-muted-foreground font-normal` (below platform name). Muted color creates de-emphasis without needing a smaller size.

### Stocks Section

Sheet fields (in order):
1. Label: "Platform" — Input type="text", placeholder="e.g. Zerodha"
2. Label: "Current Value (₹)" — Input type="text" inputmode="decimal", placeholder="e.g. 50,000"

List row display:
- Primary: platform name — `text-sm font-semibold`
- Secondary: current value in INR format — `text-sm text-muted-foreground font-normal`, right-aligned

### INR Bank Savings Section

Sheet fields (in order):
1. Label: "Account Label" — Input type="text", placeholder="e.g. HDFC Savings"
2. Label: "Balance (₹)" — Input type="text" inputmode="decimal", placeholder="e.g. 2,50,000"

List row display:
- Primary: account label — `text-sm font-semibold`
- Secondary: balance in INR format — `text-sm text-muted-foreground font-normal`, right-aligned

### Retirement Section (inline form)

Fields (in order):
1. Label: "NPS Current Balance (₹)" — Input type="text" inputmode="decimal"
2. Label: "EPF Current Balance (₹)" — Input type="text" inputmode="decimal"
3. Save button + inline error

Below the form, a read-only display card:
- Heading: "Projected Corpus at Retirement" — `text-sm font-semibold`
- NPS projected: "NPS ₹X" — `text-sm font-normal`
- EPF projected: "EPF ₹X" — `text-sm font-normal`
- Total projected: "Total ₹X" — `text-xl font-semibold`
- Footnote: "Based on current Settings assumptions" — `text-sm text-muted-foreground font-normal`
- If Settings retirement assumptions are missing (age = 0 or not set): show "Configure retirement assumptions in Settings to see projection." in muted text instead of the values.

### Settings Section (inline form)

Block 1 — Gold Prices:
- Heading: "Gold Prices" — `text-sm font-semibold`
- Fields: "24K price per gram (₹)", "22K price per gram (₹)", "18K price per gram (₹)" — all Input type="text" inputmode="decimal"
- Default placeholders: "e.g. 7,200" / "e.g. 6,600" / "e.g. 5,000"
- Save button (variant="default") + inline error

Separator between blocks.

Block 2 — Retirement Assumptions:
- Heading: "Retirement Assumptions" — `text-sm font-semibold`
- Fields in order: "Current Age", "Target Retirement Age", "NPS Annual Return (%)", "EPF Annual Rate (%)"
- Current Age and Target Age: Input type="text" inputmode="numeric", placeholder values: "35", "60"
- NPS Return: Input type="text" inputmode="decimal", placeholder "10"
- EPF Rate: Input type="text" inputmode="decimal", placeholder "8.15"
- Save button (variant="default") + inline error

Separator between blocks.

Block 3 — Data:
- Heading: "Data" — `text-sm font-semibold`
- Export Data button — variant="outline" — existing implementation, keep as-is.

## Copywriting Contract

### Primary CTAs (per section)

| Section | Add CTA | Sheet Title (Add) | Sheet Title (Edit) |
|---------|---------|-------------------|--------------------|
| Gold | "Add Item" | "Add Gold Item" | "Edit Gold Item" |
| Mutual Funds | "Add Platform" | "Add MF Platform" | "Edit MF Platform" |
| Stocks | "Add Platform" | "Add Stock Platform" | "Edit Stock Platform" |
| Bank Savings | "Add Account" | "Add Bank Account" | "Edit Bank Account" |
| Retirement | — (inline Save) | — | — |
| Settings | — (inline Save per block) | — | — |

Save button label: "Save" (all forms).
Delete button label: "Delete" (Sheet footer, edit mode only).

### Empty States

| Section | Heading | Body |
|---------|---------|------|
| Gold | "No gold items yet" | "Add your first item to track your gold holdings." |
| Mutual Funds | "No platforms added" | "Add a platform to track your mutual fund investments." |
| Stocks | "No platforms added" | "Add a platform to track your stock portfolio." |
| Bank Savings | "No accounts added" | "Add an account to track your INR savings." |

Empty state visual: Centered inside Card, `py-12 text-center`. Heading: `text-sm font-semibold text-foreground`. Body: `text-sm text-muted-foreground font-normal mt-1`.

### Error States

| Trigger | Error copy |
|---------|-----------|
| Save fails (persistence error) | "Could not save. Check that the app is running and try again." |
| Required field empty on submit | Field-level: "This field is required." — shown below the specific Input |
| Invalid number (non-numeric input) | Field-level: "Enter a valid number (e.g. 1,50,000)." |
| Gold: grams ≤ 0 | Field-level: "Weight must be greater than 0." |
| Age out of range (< 1 or > 100) | Field-level: "Enter a valid age between 1 and 100." |
| Target age ≤ current age | Field-level: "Target age must be greater than current age." |

Error placement: `text-sm text-destructive mt-1` directly below the relevant Input. Persistence error: `text-sm text-destructive mt-2` directly below the Save button. Error disappears on next save attempt.

### Destructive Actions

| Action | Trigger | Confirmation approach |
|--------|---------|----------------------|
| Delete Gold Item | Delete button in Edit Sheet | No modal. Button label: "Delete". Click immediately deletes and closes Sheet. Single-click delete is acceptable for a personal local-only app with instant data.json persistence (easy to recover by re-adding). |
| Delete MF Platform | Delete button in Edit Sheet | Same — immediate delete, no modal. |
| Delete Stock Platform | Delete button in Edit Sheet | Same. |
| Delete Bank Account | Delete button in Edit Sheet | Same. |

Rationale for no confirmation modal: This is a personal local-only app. Confirmations add friction with no multi-user risk. User can re-add items quickly. This is locked per D-20 approach (no toast/modal libraries).

## Interaction States

### Button States

| State | Visual |
|-------|--------|
| Default | `variant="default"` — primary fill (near-black bg, white text) |
| Hover | Tailwind hover handled by shadcn component |
| Disabled (while saving) | `disabled` prop + `opacity-50 cursor-not-allowed` — applied during async save |
| Loading (while saving) | Button text changes to "Saving…" while `saveData` promise is pending |

### Input States

| State | Visual |
|-------|--------|
| Default | `border-input` ring |
| Focus | `ring-2 ring-ring` — shadcn default |
| Error | Add `aria-invalid="true"` + `border-destructive` class |
| Disabled | Not used in Phase 2 |

All currency inputs: `type="text" inputmode="decimal"`. Never `type="number"`. Accepts Indian formatting "1,50,000". Parsed via `parseFinancialInput()` from `src/lib/financials.ts`.

### Sheet Interaction

- Open: click "Add [Item]" button or click any list row.
- Close: Sheet overlay backdrop click, Escape key (shadcn Sheet default behavior), or after successful save.
- On successful save: close Sheet immediately. Do not show a success toast.
- On save error: keep Sheet open, show inline error below Save button.
- Delete: closes Sheet after deletion (no error expected for local delete).

## Accessibility

- All form inputs paired with `<Label>` via `htmlFor` / `id`.
- Sheet uses `role="dialog"` and `aria-labelledby` pointing to SheetTitle (shadcn default).
- Error messages use `role="alert"` so screen readers announce them on appearance.
- Delete button: `aria-label="Delete [item name]"` to distinguish from generic "Delete".
- List rows that open a Sheet: `role="button"` or wrap in `<button>` element with `aria-label="Edit [item name]"`.
- Section total: wrapped in `<output>` element with `aria-live="polite"` so recalculations are announced.
