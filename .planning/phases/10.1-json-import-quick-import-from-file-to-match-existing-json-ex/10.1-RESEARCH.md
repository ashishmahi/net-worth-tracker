# Phase 10.1 — Research

**Phase:** 10.1 — JSON import  
**Date:** 2026-04-26

## RESEARCH COMPLETE

### Question

What is needed to implement file-based JSON import that mirrors boot-time load and persists via `saveData`?

### Findings

1. **Single validation path:** `AppDataContext` already runs `migrateLegacyBankAccounts` then `DataSchema.safeParse` in the `useEffect` that loads `/api/data`. Import must use the **same** sequence on user-provided `unknown` (post-`JSON.parse`) so hand-edited exports and legacy `balanceInr` rows behave identically to a cold start.

2. **Exporting parse logic:** `migrateLegacyBankAccounts` is module-private today. A named export `parseAppDataFromImport(raw: unknown)` (or equivalent) that returns `success` + `AppData` or a failed Zod result avoids duplicating that logic in `SettingsPage`.

3. **Persistence:** `saveData` already applies optimistic context update and reverts the React state on failed `POST` / network error, matching IMP-02 (no silent partial write). Import calls `saveData(validatedData)` only after the user confirms in `AlertDialog`.

4. **UI fit:** `SettingsPage` has `handleExport`, a **Data** block (single `Export Data` button), and a Phase 9 **Clear all data** flow with `AlertDialog`, `Loader2`, and `role="alert"` errors. Import adds a hidden `input type="file"`, a second `outline` button, a confirm `AlertDialog` (after validation only), and inline `text-muted-foreground` success, matching `10.1-CONTEXT` and `10.1-UI-SPEC.md`.

5. **JSON.parse failures:** If the file is not valid JSON, we fail before Zod and show a clear message (no import dialog).

6. **Dependencies:** Phase 10 (net worth history in schema) is not shipped yet; current `DataSchema` has no `netWorthHistory` field. This implementation targets the **live** schema; when Phase 10 lands, re-run a quick regression: export → import round-trip with the new field.

## Validation Architecture

- **Build gate:** `npm run build` (tsc + vite) after changes.
- **Lint:** `npm run lint`.
- **No automated E2E** in repo today; behavior verification is **manual** (pick file, invalid file, confirm/cancel, save failure with dev server stopped).
- **Rationale:** Import is `SettingsPage` + context wiring; no test runner is configured; Nyquist coverage for this phase is build + lint + manual UAT scripts in PLAN.

## Risks / mitigations

| Risk | Mitigation |
|------|------------|
| Large file DoS in browser | Optional file size pre-check; not required for v1.3 (local use). |
| PII in console | Validation errors for users use trimmed Zod hint, not `console` dump. |
