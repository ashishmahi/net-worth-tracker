## Color

All colors are CSS variable-based (shadcn zinc preset). No hex values hardcoded in components.

| Role | CSS Variable | Light Mode Resolved | Usage in Phase 20 |
|------|-------------|--------------------|--------------------|
| Dominant (60%) | `--background` | hsl(0 0% 100%) white | Page background, card surfaces |
| Secondary (30%) | `--card` / `--muted` | hsl(0 0% 100%) / hsl(240 4.8% 95.9%) | Card container, muted text backgrounds |
| Accent (10%) | `--primary` | hsl(240 5.9% 10%) near-black | Filled "Decrypt" Button (primary variant), focus ring |
| Destructive | `--destructive` | hsl(0 84.2% 60.2%) red | Inline error text only ŌĆö `text-destructive` class |
| Muted foreground | `--muted-foreground` | hsl(240 3.8% 46.1%) grey | Eye/EyeOff icon default state, placeholder text, hint text |

Accent reserved for:
- The "Decrypt" button (primary Button variant ŌĆö used because it initiates the security-critical decrypt action)
- Focus ring on both passphrase Input fields

The "Export Data" and "Import from JSON" buttons remain `variant="outline"` (existing pattern, not accent).

Destructive color is exclusively for inline error `<p>` elements (`text-destructive`). No button in this phase uses `variant="destructive"`.

## Interaction States

### Export Block

| State | Visual | Trigger |
|-------|--------|---------|
| Idle, passphrase empty | Passphrase input blank, "Export Data" button outline/enabled | Default |
| Idle, passphrase filled | Passphrase input shows dots (password masked), eye icon visible | User typing |
| Show passphrase | Input switches to `type="text"`, EyeOff icon shown | Eye button click |
| Busy (encrypting ~250ms) | "Export Data" button disabled + Loader2 spinner, input disabled | Click with non-empty passphrase |
| Error | `<p role="alert" className="text-sm text-destructive">` below button | encryptData throws |
| Success | File download auto-triggers, passphrase field cleared to `''` | encrypt + Blob success |

### Import Block ŌĆö Encrypted Path

| State | Visual | Trigger |
|-------|--------|---------|
| No file selected | Standard Import button, no passphrase block | Default |
| Plain JSON selected | Existing AlertDialog path (unchanged) | File picker, non-encrypted file |
| Encrypted file selected | Inline decrypt block appears below Import button (`pendingEncryptedEnvelope !== null`) | File picker, `encrypted: true` detected |
| Decrypt passphrase empty | "Decrypt" button enabled (allow attempt), passphrase input blank | Default in decrypt block |
| Decrypt busy | "Decrypt" button disabled + Loader2 spinner, passphrase input disabled | Decrypt click |
| Decrypt success | Inline block unmounts (pendingEncryptedEnvelope reset), AlertDialog opens | decryptData + parse success |
| Decrypt error (wrong passphrase) | `<p role="alert">` with specific message, passphrase field stays | isCryptoError(e) |
| Decrypt error (other) | `<p role="alert">` with generic message | non-CryptoError caught |
| New file picked | Decrypt block resets (passphrase cleared, error cleared, envelope cleared) | New file selection |
| AlertDialog cancelled | pendingEncryptedEnvelope reset, importDecryptPassphrase cleared | onOpenChange(false) |

## Layout Specification

Phase 20 modifies only the existing "Data" block within SettingsPage, between the live-rates Card and the Danger Zone Separator. No new pages, no new routes, no new Card containers.

### Export Sub-block (within existing Data section)

```
[Label: "Passphrase (optional)"]
[Input type=password, placeholder="Leave blank for plain JSON", pr-10]
  ŌööŌöĆ [Eye/EyeOff absolute right-2 top-1/2]
[Button variant="outline" full-auto-width: "Export Data"]
[<p role="alert" text-sm text-destructive>]  ŌåÉ only when exportError
```

Layout: `space-y-2` vertical stack. Export passphrase sits above the Export Data button. No horizontal grouping with the Import button ŌĆö both remain in the existing `flex flex-wrap gap-2` row for the buttons; the passphrase field is inserted above that row.

Revised Data section structure:

```
<div> {/* Block 3: Data */}
  <p text-sm font-semibold>Data</p>

  {/* Export passphrase (new) */}
  <div space-y-2 mb-4>
    <Label>Passphrase (optional)</Label>
    <div relative>
      <Input type=password pr-10 ... />
      <button absolute eye-toggle />
    </div>
  </div>

  {/* Existing button row (modified: Export Data now calls handleExportClick) */}
  <div flex flex-wrap gap-2>
    <Button variant=outline: "Export Data" onClick=handleExportClick>
    <Button variant=outline: "Import from JSON">
  </div>

  {/* Export error (new) */}
  {exportError && <p role=alert text-sm text-destructive>}

  {/* Existing import errors (unchanged) */}
  {importParseError/importValidationError && ...}

  {/* Import decrypt block (new, conditional) */}
  {pendingEncryptedEnvelope && (
    <div mt-4 space-y-2>
      <Label>Passphrase</Label>
      <div relative>
        <Input type=password pr-10 ... />
        <button absolute eye-toggle />
      </div>
      {importDecryptError && <p role=alert text-sm text-destructive>}
      <Button variant=default: "Decrypt">
    </div>
  )}

  {/* Existing AlertDialog (unchanged) */}
  <AlertDialog ...>
</div>
```

Spacing note: `mt-4` (16px) is the authoritative top margin for the inline decrypt block. The RESEARCH.md code example shows `mt-3` (12px) ŌĆö that value is incorrect and must not be followed. Use `mt-4` as specified here.

## State Variable Naming

Following the established `xBusy` / `xError` per-block pattern:

| Variable | Type | Rationale |
|----------|------|-----------|
| `exportPassphrase` | `string` | Controlled input value, export passphrase field |
| `showExportPassphrase` | `boolean` | Eye toggle state for export field |
| `exportBusy` | `boolean` | Disable + spinner during encryptData call |
| `exportError` | `string \| null` | Inline error below Export Data button |
| `pendingEncryptedEnvelope` | `EncryptedEnvelope \| null` | Stores parsed envelope when encrypted file detected |
| `importDecryptPassphrase` | `string` | Controlled input value, import decrypt field |
| `showImportDecryptPassphrase` | `boolean` | Eye toggle state for import decrypt field |
| `importDecryptError` | `string \| null` | Inline error below Decrypt button |

`importBusy` (existing) is reused for the Decrypt button busy state. No second busy variable needed ŌĆö decrypt and save are sequential, not overlapping.
