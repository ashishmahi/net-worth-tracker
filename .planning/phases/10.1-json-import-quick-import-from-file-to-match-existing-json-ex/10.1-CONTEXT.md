# Phase 10.1: JSON import - Context

**Gathered:** 2026-04-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Users can **choose a local JSON file** in the app (Settings **Data** section, next to **Export**), the app **reads** the file and runs the **same** `migrateLegacyBankAccounts` + **`DataSchema.safeParse`** path as initial load, then on success **persists** the full document via **`saveData`** (POST `/api/data`). Invalid files or failed saves surface **clear errors** with **no** silent partial write. **Out of scope:** new API routes, multi-file merge, import preview/edit UI beyond confirm-before-save, chart work (Phase 11).

**Requirements:** **IMP-01**, **IMP-02** in `.planning/REQUIREMENTS.md` ‚Äî this context locks **how**; those reqs lock **what**.

</domain>

<decisions>
## Implementation Decisions

### A ‚Äî Confirm before replace (IMP-02)
- **D-01:** After the user picks a file, **if** parse + migration + `DataSchema` succeed, show a **short shadcn-style `AlertDialog`**: title + 1‚Äì2 lines that the action **replaces** current in-memory and on-disk wealth data, with **Cancel** and a **continue** (confirm) action, then call **`saveData`**. If validation **fails**, **do not** show this dialog ‚Äî only error feedback.
- **D-02:** Dialog is the **import** safety step (lighter than ‚ÄúClear all data‚Äù but same family of explicit confirmation for destructive *replace* intent).

### B ‚Äî Placement, control, and success feedback
- **D-03:** **Data** block: place **Import** on the **same row** as **Export Data**, with responsive **wrap** on narrow widths (user preference: same row).
- **D-04:** **Hidden** `<input type="file" accept="application/json,.json" />` (or appropriate `accept`) triggered by an **Import** button (`variant="outline"` to align with Export‚Äôs outline style unless implementation finds a contrast issue).
- **D-05:** On **success**, show a **inline muted** status line in the Data block (pattern parallel to post-clear: *"All data has been cleared"*) ‚Äî e.g. that data was imported from file; **Claude‚Äôs discretion:** exact string, persistence until navigation vs next import attempt.
- **D-06:** **Busy state:** while reading file / validating, disable repeat triggers and show existing pattern (**`Loader2`**) as elsewhere on Settings, **Claude‚Äôs discretion:** per-button vs inline spinner.

### C ‚Äî Invalid file and save errors
- **D-07:** **Invalid or non-matching data:** a **friendly** primary line (file isn‚Äôt valid app data / doesn‚Äôt match this app‚Äôs format) **plus a short technical hint** for people fixing files: e.g. first Zod path/issue or a compact count of issues ‚Äî **not** a full error dump. **Claude‚Äôs discretion:** format (single line vs two lines).
- **D-08:** **Save failure** after user confirms: match **Settings** error pattern ‚Äî `role="alert"`, same **tone** as other blocks, e.g. *‚ÄúCould not save imported data. Check that the app is running and try again.‚Äù* (align with **Phase 9** / clear-data failure style).

### D ‚Äî Scope and edge cases
- **D-09:** **In scope** for success: any JSON that becomes valid **`AppData`** after the **same** path as app boot (includes hand-edited files if they parse and validate); product **copy** may still say an export is the **expected** source without blocking hand edits.
- **D-10:** A file that validates to an **‚Äúempty‚Äù** or initial-like slate (e.g. `createInitialData()`-shaped) is **allowed** without an extra ‚Äúempty import‚Äù warning ‚Äî import is **valid**; user may intend replace-with-empty (user choice: allow).

### Claude's Discretion
- Exact **AlertDialog** strings, **aria-** attributes, and minor **Data** block layout on very small viewports.
- **Trim** Zod/parsing messages for the ‚Äúhint‚Äù line to stay short.
- Whether to **clear** success line on route change (follow Phase 9 **minimal** pattern).

</decisions>

## Existing Code Insights

### Reusable assets
- **`useAppData()`** ‚Äî `saveData`, `data`; `saveData` reverts in-memory on failed POST (no partial write).
- **`SettingsPage`** ‚Äî `handleExport` (JSON blob download); per-block error/success **inline** text; **`Loader2`**; **`AlertDialog`** in danger zone (reuse component family for import confirm).
- **`migrateLegacyBankAccounts`** + **`DataSchema`** ‚Äî must mirror **initial** `useEffect` load path so import and boot behave the same.

### Established patterns
- **Destructive / irreversible** flows use **`AlertDialog`**; **Data** copy distinguishes **Export** (backup) from **clear**; import confirm sits between export one-click and danger-zone severity.
- **Settings** errors: **`role="alert"`**, no toast library in Phase 9 decisions.

### Integration points
- Add import UI and state in **`SettingsPage`** (Data section per ROADMAP), wired to `FileReader` (or async read), then shared validation, then `saveData`.
