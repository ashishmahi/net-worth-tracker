## Phase Scope — Controls & Placement

| Surface | Control | Placement |
|---------|---------|-----------|
| Desktop | Native `<select>` | `AppTopbar` right cluster: **after** “Live prices” pill, **before** USD/INR chip (D-03) |
| Mobile | Same native `<select>` | `MobileTopBar` — fit with existing 44px touch targets; may wrap row without dropping min tap sizes |

**Options (fixed six):** INR, USD, AED, EUR, GBP, SGD — option **labels** `symbol + code` (e.g. `₹ INR`, `$ USD`, `AED`) (D-04).

**Behavior:** Changing value updates `settings.reportingCurrency` via `AppDataContext`, persists to storage (RC-03). **Instant recalculation** on **Dashboard** route only for Phase 35 (net worth, category totals, breakdown rows); other routes may refresh on navigation (D-02).

**Out of scope:** Compact badge/chip + popover on mobile (`35-CONTEXT.md` deferred list).

### Visual hierarchy

- **Dashboard route:** Net worth headline, category totals, and breakdown rows remain the **primary focal area**. The reporting `<select>` is **persistent secondary chrome** (top bar), not the hero of the page.
- **Desktop top bar:** Section title + “as of” date stay left; right cluster reads left-to-right: **Live prices** pulse → **reporting currency** → USD/INR → BTC — user’s reporting lens immediately before market chips (D-03).

## Spacing Scale

Declared values (multiples of 4). New UI should use this scale; align with existing topbar clusters.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight inline gaps |
| sm | 8px | `gap-2` between Live pill, select, FX chips (matches current topbar) |
| md | 16px | Default padding inside themed wrappers |
| lg | 24px | Section-level gaps |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major breaks |
| 3xl | 64px | Page-level |

**Exceptions:**

- Existing `AppTopbar` uses `py-[18px]` and `px-8` — **do not refactor** in this phase; new control sits in the existing flex row with `gap-2` (8px).
- Native `<select>` minimum tap height **≥ 44px** on mobile (match `MobileTopBar` buttons).

## Typography

Max **4** sizes and **2** weights (400 / 600) per contract.

| Role | Size | Weight | Line height | Usage |
|------|------|--------|-------------|-------|
| Label | 14px (`text-sm`) | 600 | 1.4 | Accessible name / compact labels near control |
| Body | 16px (`text-base`) | 400 | 1.5 | Default page copy (unchanged) |
| Heading | 20px (`text-xl`) | 600 | 1.25 | In-page headings (`PageHeader` pattern) |
| Compact meta | 12px (`text-xs`) | 400 | 1.45 | Topbar meta alignment where matching pills |

**Note:** Current `AppTopbar` title uses 13px — new select label styling should **harmonize** with adjacent `text-[11.5px]` pills without introducing a fifth contract size; prefer `text-xs` / `text-sm` tokens only.

## Color

Semantic roles reference **`src/index.css`** HSL tokens (light/dark). No new palette.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` | App shell / header backdrop |
| Secondary (30%) | `--card`, `--muted` | Chips, select surface, borders |
| Accent (10%) | `--primary`, `--ring` | Focus ring on `<select>`; active nav elsewhere — **not** filling entire topbar |
| Destructive | `--destructive` | Destructive actions only (none in this phase) |

**Accent reserved for:** focus-visible ring on the reporting `<select>`, primary sidebar/nav emphasis (existing patterns), chart accents — **not** “all interactive elements.”

**60/30/10:** Maintain existing shell balance: large background, card-like chips/select, sparse primary/focus.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary user action (this phase) | **Choose reporting currency** — user picks one of six options in the topbar; amounts on the Dashboard update immediately; choice persists after reload. |
| Control accessible name | **Reporting currency** (`aria-label` or associated `<label>`), screen-reader first. |
| Empty state (selector) | **N/A** — six options always present. |
| Empty state (dashboard) | No new copy — keep existing `DashboardPage` empty and onboarding patterns when the user has no tracked holdings. |
| Error / degradation (FX) | When a rate is missing (**FX-03**): show **Rate unavailable** next to affected amounts; keep original currency figure; do not crash. |
| Destructive confirmation | **None** this phase. |
