## Spacing Scale

Declared values (multiples of 4). **Inherit** Tailwind/shadcn usage on Settings; this phase does not introduce new ad-hoc pixel gaps.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), tight inline padding |
| sm | 8px | `gap-2` between row controls (Export + Import) |
| md | 16px | Default block spacing (`mb-4` on Data heading, `space-y-4` family) |
| lg | 24px | Card/section padding (`p-6` on cards elsewhere) |
| xl | 32px | Page-level vertical rhythm between main Settings sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level (rare) |

**Exceptions:** None for this phase. Spinners use `h-4 w-4` (16px) — on-scale.

## Typography

Match existing Settings: section labels and body use `text-sm`; descriptions use `text-sm text-muted-foreground`. **At most four** size roles for this feature surface:

| Role | Size | Weight | Line height | Usage |
|------|------|--------|-------------|--------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 | Dialog body, error lines, success line, hints |
| Label / section | 14px (`text-sm`) | 600 (semibold) | 1.5 | “Data” block title |
| Dialog title | 16px+ (`text-lg` or AlertDialog default title) | 600 | ~1.2 | Import replace confirmation / alert dialog title |
| Muted / secondary | 14px (`text-sm`) | 400 | 1.5 | Muted-foreground lines, `AlertDialogDescription` content |

**Weights used only:** 400, 600 (per project/shadcn defaults).

## Color

All values are **semantic** and reference existing CSS variables (`index.css`); do not introduce new hex for this phase.

| Role | Token / variable | Usage |
|------|------------------|--------|
| Dominant (~60%) | `hsl(var(--background))` | Page/Settings background |
| Secondary (~30%) | `hsl(var(--card))`, `hsl(var(--muted))`, `hsl(var(--border))` | Surfaces, separators, `outline` button chrome |
| Accent (~10% reserved) | `hsl(var(--primary))`, `hsl(var(--ring))` | **Not** the default for Import/Export (those stay `variant="outline"`). Use `primary` / `ring` for default primary actions and focus only — consistent with app shell. |
| Destructive | `hsl(var(--destructive))` + foreground | Error text `text-destructive` (`role="alert"`), not for Import trigger |

**Accent reserved for:** default primary `Button` elsewhere, focus ring on interactive elements, `sidebar-primary` accent where used — **not** for the Data row’s Export/Import pair (both outline).

**Interaction states:** Errors: `text-destructive` + `role="alert"`. Success line after import: `text-sm text-muted-foreground` (same family as “All data has been cleared.”). Busy: `Loader2` inherits foreground color, `aria-hidden` on icon.

## Copywriting Contract

| Element | Copy (contract) |
|---------|------------------|
| Import trigger (button) | **“Import from JSON”** (verb + object; `outline` to match **Export Data** row) |
| `aria-label` (recommended) | “Import wealth data from a JSON file” (or equivalent explicit phrase) |
| File input | Hidden; `accept` includes `application/json` and `.json` (implementation detail) |
| Replace confirmation — title | **“Replace with imported data?”** (or “Import this file?” — must name replace intent) |
| Replace confirmation — body (1–2 short lines) | State that the action **replaces** current in-memory and on-disk wealth data with the file contents; no preview beyond this dialog. |
| Confirm (primary action in dialog) | **“Import and replace”** or **“Yes, import”** — not generic “OK”/“Save” |
| Cancel | **“Cancel”** (AlertDialogCancel) |
| Validation / parse failure (friendly) | **Primary line:** the file is not valid app data or does not match the expected format. **Secondary (hint):** one short line — first Zod path/issue or a compact count — not a full dump. |
| Save failure after user confirmed | **“Could not save imported data. Check that the app is running and try again.”** (align **Phase 9** / clear-data failure tone) |
| Success (inline muted, Data block) | Short confirmation that data was imported (exact string implementer’s choice; pattern parallel to *“All data has been cleared.”* — `text-muted-foreground`, `role="status"` as appropriate) |
| Empty state | **N/A** — this phase extends the existing **Data** block; there is no separate empty “list” screen. |

**Destructive / irreversible note:** Per CONTEXT, the import confirm is a **replace** (same family as clear-data, lighter severity) — `AlertDialog` with explicit confirm; **if validation fails, do not open** the dialog (errors only).

## Layout & visual hierarchy (Phase 10.1)

| Focal / order | Description |
|---------------|-------------|
| Data block | User sees **Data** label first, then **one row** (or wrapped row): **Export Data** + **Import from JSON** with `gap-2` / `flex flex-wrap` — **same row** on wide viewports, wrap on narrow. |
| Modal (when open) | `AlertDialog` is the only focal layer; title then description then footer actions. |
| Busy | Disable repeat of import/export while reading/validating/saving; `Loader2` (existing Settings pattern) on or beside the in-flight control. |

**Accessibility (contract):** Hidden file input is triggered only by the visible `Import from JSON` button. Dialogs use Radix `AlertDialog` with descriptive title; errors use `role="alert"`. Exact `aria-*` left to implementation (CONTEXT “Claude’s discretion”).
