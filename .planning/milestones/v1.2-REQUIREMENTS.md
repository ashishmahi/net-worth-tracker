# Requirements — v1.2 Data reset & clean slate

**Milestone:** v1.2  
**Goal:** User can **clear all wealth data** and return to a **valid empty state** (same as a fresh `INITIAL_DATA` load) after an **unmissable warning** and **non-accidental confirmation** — not one stray click.  

**Previous / shipped:** v1.1: [`.planning/milestones/v1.1-REQUIREMENTS.md`](v1.1-REQUIREMENTS.md)  

**Research (lightweight):** `.planning/research/SUMMARY.md` — in-repo patterns only.  

---

## Active requirements (v1.2) — **COMPLETE**

### Data — clear all

- [x] **DATA-01**: User can start a “clear all data” / “reset app data” flow from a **discoverable** location (e.g. **Settings**, separated visually as a “Danger” or data-management block so it is not confused with normal saves). *Outcome: Settings “Danger zone” `Card` below **Export** with **Clear all data**.*  
- [x] **DATA-02**: Before any delete runs, the UI shows a **clear warning** that: (a) the action is **irreversible**, (b) **all** net-worth and asset rows stored in `data.json` will be removed, (c) optional backup is *your* existing JSON export in Settings. The user must **confirm** in a way that **cannot** happen from a single accidental click. *Outcome: **AlertDialog** with **Cancel** + distinct destructive **Yes, clear all data**; copy references **Export Data** above.*  
- [x] **DATA-03**: On confirmed reset, the app **persists** a document equivalent to **`INITIAL_DATA`** (via **`createInitialData()`** / **`saveData` / POST** `/api/data`**) and **in-memory** context matches. *Outcome: full `DataSchema` empty document, `saveData` rollback on error; RHF re-sync when optional `settings` keys removed.*
