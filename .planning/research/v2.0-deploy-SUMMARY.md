# v2.0 — Deploy research summary

## Stack / tooling

- **Docker:** Multi-stage build — Node image runs `npm ci` + `npm run build`; final stage **nginx:alpine** (or distroless static) serves `dist/`. SPA routing: `try_files $uri $uri/ /index.html` so refreshes on nested paths do not 404 (this app uses in-app section state, not URL routes; still safe for future deep links).
- **GitHub Pages:** Project Pages URL is `https://<user>.github.io/<repo>/` → Vite needs **`base: '/<repo>/'`** (or env-driven). Use **`import.meta.env.BASE_URL`** for any manual asset paths; Vite injects base into built assets.
- **GitHub Actions:** `actions/checkout`, `actions/setup-node` with npm cache, run `npm ci`, `npm test`, `npm run build`. Deploy: **peaceiris/actions-gh-pages** (push `dist` to `gh-pages` branch) or **`actions/upload-pages-artifact`** + **`actions/deploy-pages`** (modern Pages API). Requires repo **Settings → Pages → Source**: GitHub Actions or `gh-pages` branch.

## Pitfalls

- **Wrong `base`** → blank page or broken JS/CSS on Pages.
- **Secrets:** Default **`GITHUB_TOKEN`** suffices for Pages deploy to same repo; document fork behavior.
- **Private data:** Hosted demo is still **client-only localStorage** — beta banner should remind users data stays in browser / device.
