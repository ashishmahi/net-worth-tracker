```typescript
/** v1.5 files without `liabilities`: default to `[]` before Zod parse. */
export function ensureLiabilities(raw: unknown): unknown {
  if (raw === null || typeof raw !== 'object') return raw
  const o = raw as Record<string, unknown>
  if (!('liabilities' in o) || o.liabilities === undefined) {
    return { ...o, liabilities: [] }
  }
  return raw
}
```

Key differences from `ensureOtherCommodities`:
- Checks root `o` directly (NOT `o.assets`) because `liabilities` is a root-level key (D-06)

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-14-02 | Information Disclosure | liabilities array in data.json | accept | Local-only app, no network exposure; data.json is user's own file on local filesystem |
