**Core value:** Total net worth in INR with live prices where applicable; gold uses the same fetch + context pattern as silver; Settings commodity pricing UX parity for gold and silver.

## v2.0.1 Requirements (final)

### Live spot (API & context)

- [x] **SPOT-01**: App fetches **gold spot USD per troy ounce** from **gold-api.com** (e.g. **`/price/XAU`**) with response validation **consistent** with `fetchSilverUsdPerOz` (non-positive / shape errors throw).  
- [x] **SPOT-02**: Client TTL for gold matches the **silver** channel (`GOLD_TTL_MS` same as `SILVER_TTL_MS` unless a comment documents an exception).  
- [x] **SPOT-03**: `LivePricesContext` exposes **`goldUsdPerOz`**, **loading**, **error**, and includes gold in **`refetch`** and the same **stale-refresh** strategy as silver (mount, interval, visibility).  

### Product UX (hints)

- [x] **UX-01**: When **gold spot** and **USD→INR** are available, the UI shows **read-only live ₹/gram hints** for **24K, 22K, and 18K**, derived from spot using **`TROY_OZ_TO_GRAMS`** and standard purity ratios (**22/24**, **18/24**). Primary placement: **Settings → Gold Prices**; **Gold** page may repeat hints for consistency.  
- [x] **UX-02**: If spot or forex is missing or errored, the UI shows **loading / error** affordances **consistent with** existing live-price patterns (e.g. silver on Commodities, Session rates on Settings).  
- [x] **UX-03**: Persisted **`settings.goldPrices`** remains under user control via **`goldPricesLocked`** and **`GoldSpotPricesSync`** when unlocked (see phase docs).  

### Settings layout parity (Phase 27)

- [x] **UX-04**: **Settings** uses a **shared interaction pattern** for **gold** and **silver** unit pricing: compact display by default when live quotes define effective prices.  
- [x] **UX-05**: When the relevant **live** data for that metal (and **USD→INR** where needed) is **available and not in an error state**, the pricing block is **read-only** until the user chooses **Edit**; **Edit** reveals inputs **prefilled** from **fetched** values (and any persisted overrides in effect).  
- [x] **UX-06**: When **live fetch fails** or **required** spot/forex inputs are **missing**, the same block defaults to **editable** (user is not forced to click **Edit** to type a price).  
- [x] **UX-07**: **Silver** has **Settings** parity with gold — visible **₹/g**, **Edit**, persisted **`silverInrPerGram`** / **`silverPricesLocked`**, **`SilverSpotPricesSync`**, effective silver in **`dashboardCalcs`**.  

### Calculations & tests

- [x] **CALC-01**: Shared helper(s) compute **INR per gram** from **USD/oz** and **`usdInr`** for each karat, **rounded** consistently with `roundCurrency` / existing commodity math.  
- [x] **TEST-01**: Unit tests cover **gold API parse**, **derivation math**, and related exports used by dashboard or Settings.  

## Future requirements (deferred)
