<threat_model>
## Trust Boundaries

- **Client-only:** Passphrases and plaintext exist only in browser memory; zip bytes are user-local.
- **Files:** Only **`data.json`** inside user-supplied zip is parsed as wealth data after optional decryption.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | Information disclosure | Passphrase in UI | mitigate | No logging of passphrase or zip plaintext; password inputs default hidden + Eye toggle |
| T-21-02 | Tampering | Malicious zip | mitigate | Single entry **`data.json`**; parse only after successful extract; **`parseAppDataFromImport`** gate unchanged |
| T-21-03 | Denial of service | Large zip | accept | Same class as pre-phase large JSON; optional future size cap out of scope |
| T-21-04 | Spoofing | Wrong file type | mitigate | Reject non-zip at UX boundary with explicit message; no silent JSON import |

</threat_model>
