    1. Define **`const WEALTH_STORAGE_KEY = 'wealth-tracker-data'`** at module scope.

    2. **Remove** the **`useEffect`** that **`fetch('/api/data')`**.

    3. Add **`function readInitialWealthState(): { data: AppData; loadError: string | null }`** at module scope (or immediately above the provider). Implementation:
       - **`raw = localStorage.getItem(WEALTH_STORAGE_KEY)`**
       - If **`null`** or **`''`** → **`{ data: createInitialData(), loadError: null }`**
       - Else try **`JSON.parse(raw)`** — on catch → **`{ data: createInitialData(), loadError: 'Could not read stored data. Starting with defaults.' }`** (exact string negotiable but must not mention **`data.json`** or a server)
       - On successful parse, **`parseAppDataFromImport(parsed)`** — if **`success`** → **`{ data: result.data, loadError: null }`**; else → **`{ data: createInitialData(), loadError: 'Saved data format is unrecognized. Starting with defaults to avoid data loss.' }`**; **`console.warn`** must not say **`data.json`** (use **`stored data`** or **`schema`**).

    4. **`useState(() => readInitialWealthState().data)`** and **`useState(() => readInitialWealthState().loadError)`** — duplicate **`getItem`** is acceptable (two identical reads). Alternatively merge into one **`useState`** holding **`{ data, loadError }`** if you prefer single updates — either pattern is valid if context API **`value={{ data, saveData, loadError }}`** unchanged.

    5. **`saveData(newData)`:** Drop **`fetch`**. **`try { localStorage.setItem(WEALTH_STORAGE_KEY, JSON.stringify(newData)); setData(newData) }`** — on **`QuotaExceededError`** (or any throw), **`setData(previous)`** if optimistic update was applied, then **`throw err`**. CONTEXT allows dropping the old **`previous`** snapshot pattern only if ordering avoids inconsistent UI — minimum: **must throw** on quota so callers show STORE-05 errors.

    6. **Never** **`localStorage.clear()`**.

    7. Update comments that refer to **`data.json`** as the storage medium.
