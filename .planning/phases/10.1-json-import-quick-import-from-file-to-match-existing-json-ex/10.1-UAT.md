## Tests

### 1. Data block — Import beside Export
expected: |
  Open Settings. In the **Data** section you see **Export Data** and **Import from JSON** on the same row (on a wide viewport) or wrapped on a narrow one. Both are outline-style buttons. A file input is not visible.
result: pass

### 2. Valid JSON — confirm and persist
expected: |
  (a) **Export** wealth JSON to a file. (b) **Import** that file. After the file is read, a **Replace with imported data?** dialog appears (not before). (c) Click **Import and replace**. A muted success line appears (e.g. wealth data imported). (d) Reload the app — your data still matches the imported file (no loss).
result: pass

### 3. Invalid file — not JSON
expected: |
  Choose a file that is not valid JSON (e.g. plain text). You get a clear error. The **replace** dialog does **not** open. Current in-app data is unchanged.
result: pass

### 4. Valid JSON — invalid app data
expected: |
  Choose a file with valid JSON but not valid `AppData` (e.g. empty `{}` or delete required keys from an export). You see a friendly first line and a second line with a short hint (Zod path or count). The replace dialog does **not** open.
result: pass

### 5. Cancel on replace dialog
expected: |
  Choose a **valid** exported file so validation succeeds and the **Replace with imported data?** dialog opens. Click **Cancel**. The dialog closes with **no** success line and your previous wealth data in the app is unchanged (e.g. compare before/after total or reload).
result: pass

### 6. Save failure (dev server off)
expected: |
  With the Vite dev server **stopped** (or `/api/data` otherwise failing), pick a valid file, confirm **Import and replace**. You see **Could not save imported data. Check that the app is running and try again.** (or equivalent) with alert semantics; in-memory state should not show a false success.
result: pass
