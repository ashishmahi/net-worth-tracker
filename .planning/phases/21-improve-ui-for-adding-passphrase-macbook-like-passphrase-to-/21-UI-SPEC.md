## Color

All colors via CSS custom properties defined in `src/index.css`. No raw hex values — always use the semantic token.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` = white (#ffffff) | Page surface, dialog background |
| Secondary (30%) | `hsl(var(--card))` / `hsl(var(--muted))` = zinc-100 equiv | Cards, input backgrounds, secondary buttons |
| Accent (10%) | `hsl(var(--primary))` = zinc-900 (#18181b equiv) | Primary action buttons only (Export zip, Decrypt, Import and replace) |
| Destructive | `hsl(var(--destructive))` = red-500 | Error text (`text-destructive`), destructive button variants |

Accent reserved for:
- The "Export" button inside the export passphrase modal
- The "Decrypt" button inside the import passphrase modal
- The "Import and replace" button inside the import confirmation dialog

Destructive (`text-destructive`) reserved for:
- Inline error messages inside all modals: `<p role="alert" className="text-sm text-destructive">`
- "Clear all data" button (existing, unchanged)

## Interaction Contract

### Export Flow

1. Single "Export Data" button on SettingsPage (variant: `outline`). Replaces current split button + inline passphrase field.
2. Clicking "Export Data" opens the Export Passphrase AlertDialog immediately (D-03).
3. Export Passphrase AlertDialog contains:
   - Title: "Export data"
   - Description: helper text below title
   - Passphrase `Input` (type="password", `autoComplete="off"`) with `Label` "Passphrase"
   - Show/hide toggle (ghost Button, `Eye`/`EyeOff`, positioned absolute right inside Input)
   - Hint text below input: "Leave blank to export without a passphrase" (`text-sm text-muted-foreground`)
   - Inline error area below hint: `<p role="alert" className="text-sm text-destructive">` — visible only on encryption failure
   - Footer: `AlertDialogCancel` "Cancel" + primary `Button` "Export" (disabled + `Loader2` spinner while `exportBusy`)
4. On success: modal closes, zip download triggers. No inline success state needed on page.
5. On cancel: modal closes, passphrase state cleared.

### Import Flow

1. "Import from Zip" button on SettingsPage (variant: `outline`). Hidden `<input type="file" accept=".zip">` (changed from `.json`).
2. On file selection:
   - If file is not a zip → show inline error below Import button on the page (not inside a modal): "This file format is no longer supported. Please re-export from the app."
   - If zip is unencrypted → proceed directly to existing import confirmation AlertDialog (no change).
   - If zip is password-protected → Import Passphrase AlertDialog opens automatically (D-08, D-09).
3. Import Passphrase AlertDialog contains:
   - Title: "This file is password protected"
   - Description: helper text
   - Passphrase `Input` (type="password", `autoComplete="off"`) with `Label` "Passphrase"
   - Show/hide toggle (same pattern as export modal)
   - Inline error inside dialog: `<p role="alert" className="text-sm text-destructive">`
     - Wrong passphrase: "Wrong passphrase — the file could not be decrypted."
     - Other failure: "Decryption failed. Check that the app is running and try again."
   - Footer: `AlertDialogCancel` "Cancel" + primary `Button` "Decrypt" (disabled + `Loader2` while busy)
4. On successful decrypt: passphrase modal closes → existing import confirmation AlertDialog opens (D-10).
5. On cancel or new file pick: passphrase modal state fully cleared (D-11).

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Error messages announced to screen readers | `<p role="alert" className="text-sm text-destructive">` — all inline errors |
| Passphrase input announce state | `aria-label` or associated `Label` with `htmlFor` |
| Show/hide toggle | `aria-label="Show passphrase"` / `"Hide passphrase"` on ghost Button |
| Loading state | Export/Decrypt button: `disabled` attribute + visual spinner; no separate `aria-busy` needed (button text changes) |
| Dialog focus trap | Provided by shadcn AlertDialog / Radix Dialog (no extra work) |
| Dialog close on Cancel | `AlertDialogCancel` handles Escape + click (Radix built-in) |
