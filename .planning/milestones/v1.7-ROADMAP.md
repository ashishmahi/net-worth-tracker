## v1.7 — localStorage Migration (Phase 22)

**Goal:** The app persists all user data in browser `localStorage` — no Vite dev-server plugin, no `data.json` persistence API, no `fetch` calls for wealth persistence.

| Phase | Name | Goal (summary) |
|-------|------|----------------|
| **22** | localStorage Migration | `AppDataContext` reads/writes key **`wealth-tracker-data`**; sync boot; remove **`plugins/dataPlugin.ts`**; Settings + CLAUDE copy; Vitest + **happy-dom** |

### Phase 22 — plans (1)

- [x] [22-01-PLAN.md](../phases/22-localstorage-migration/22-01-PLAN.md) — localStorage boot/save, infra removal, UX copy, tests  

**Success criteria** (from roadmap):

1. App loads saved data immediately on refresh — synchronous boot (no flash-of-empty-state).
2. Edits persist across sessions without server or repo `data.json`.
3. Theme key survives saves (`setItem` only — never `clear()`).
4. User-facing copy references browser storage, not disk `data.json`.
5. Vitest passes including boot-path coverage.
