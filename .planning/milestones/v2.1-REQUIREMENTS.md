# Requirements archive — Personal Wealth Tracker — Milestone v2.1

**Archived:** 2026-05-04 (milestone close). There was no live `.planning/REQUIREMENTS.md` at close (removed at **v2.0.1**); this archive captures **v2.1** routing goals from **Phase 28** planning and verification.

**Defined:** 2026-05-04  
**Core value:** Stable section URLs with **`BASE_URL`** alignment; refresh and pasted links open the correct section; mobile users can return to the dashboard without opening the drawer.

## v2.1 Requirements (final)

### Routing & navigation

- [x] **ROUTE-01**: **`BrowserRouter`** uses **`basename={import.meta.env.BASE_URL}`** (Vite **`base`** alignment for GitHub Pages).  
- [x] **ROUTE-02**: Each **`SectionKey`** maps to exactly one path; **refresh** on a section URL (e.g. **`/gold`**) keeps that section visible.  
- [x] **ROUTE-03**: **`MobileTopBar`** shows a **Home** affordance (house icon, **`aria-label="Go to dashboard"`**) when the route is **not** the dashboard index.  
- [x] **ROUTE-04**: Unknown paths **`Navigate`** to **`/`** (dashboard) with **replace** — no blank screen.  
- [x] **ROUTE-05**: Sidebar uses **`NavLink`** / URL-derived active state; active item reflects **`aria-current="page"`** where applicable.  
- [x] **ROUTE-06**: **`sectionToPath`** / **`pathToSection`** whitelist only — no navigation from arbitrary user-controlled path strings into custom HTML.  

### Verification & UAT

- [x] **UAT-01**: Manual **28-UAT.md** — deep links + refresh, mobile Home, sidebar active state — **4/4 passed** (2026-05-04).
