## Task 10.1-01-01 — Shared parse helper + import UX

<read_first>
- `src/context/AppDataContext.tsx` (full)
- `src/types/data.ts` (DataSchema)
- `src/pages/SettingsPage.tsx` (Data + Danger + export/clear)
- `.planning/phases/10.1-json-import-quick-import-from-file-to-match-existing-json-ex/10.1-UI-SPEC.md` (copy)
- `.planning/phases/10.1-json-import-quick-import-from-file-to-match-existing-json-ex/10.1-CONTEXT.md` (D-01–D-10)
</read_first>

<action>
1. In `AppDataContext.tsx`, export a function `parseAppDataFromImport(raw: unknown)` that returns a discriminated union: on success, `{ success: true; data: AppData }`; on failure, `{ success: false; zodError: z.ZodError }`. **Implementation must** call the existing `migrateLegacyBankAccounts` then `DataSchema.safeParse` in that order, mirroring the `useEffect` load path. Refactor the `useEffect` to use this helper: on success set state; on failure use existing `console.warn` + `setLoadError` pattern.

2. In `SettingsPage.tsx`, in the **Data** block: add a hidden `input` with `ref`, `type="file"`, `accept="application/json,.json"`, and `onChange` that reads the file with `file.text()` (or `FileReader` if must support very old runtimes), `JSON.parse` in try/catch (user-facing error for invalid JSON — friendly line, no dialog), then `parseAppDataFromImport`. On Zod failure: set primary error line and a **second line** with first Zod path/message or compact count (per CONTEXT D-07). On success: store parsed `AppData` in state and open a new **shadcn `AlertDialog`**, title and body per `10.1-UI-SPEC` (“Replace with imported data?”; explains replace of in-memory + on-disk; confirm `Import and replace` / `Cancel`).

3. On confirm, `await saveData(pendingData)`. In `catch`, show **could not save** copy: `Could not save imported data. Check that the app is running and try again.` (exact string per UI-SPEC) with `role="alert"`. On success: close dialog, clear pending, set inline `text-sm text-muted-foreground` `role="status"` success in Data block (parallel to “All data has been cleared.”).

4. **Busy / disable:** while reading/parsing/saving import, set busy state; disable both **Export Data** and **Import from JSON** and show `Loader2` on the in-flight path per existing patterns.

5. Re-export `parseAppDataFromImport` only from `AppDataContext` — do not add new API routes.
</action>
